1. Таблица products:
	- внести столбцы created_at и updated_at позволяющие отслеживать историю создания товара и его изменение;
	- учесть вещественное хранение цен и их правильные арифметические операции (без окгругления) в виде формата NUMERIC.
2. Таблица categories:
	- описывает возможные категории товаров из таблицы products.
3. Таблица products_categories:
	- для упрощения разработки предлагаю реализовать связанность товаров и категорий по типу ManyToOne. Для каждого товара есть лишь одна соответствующая категория. Множественное количество категорий для нас необязательно.
4. Таблица users:
	- предлагаю исключить удаление пользователя из базы данных. Лишь внести поле enabled, которая переводит соостояние аккаунта из активного в пассивное (отключенное).
5. Таблица roles:
	- описывает возможные роли пользователей. Например, superadmin (позволяет создавать новых пользователей и давать им роли), admin, manager, user.
6. Таблица users_roles:
	- связаность пользователей и их ролей предлагаю сделать по типу ManyToMany. Например, user1 может быть одновременно и админом и менеджером.
7. Таблица reviews:
	- таблица опысывает отзывы на представленные в магазине товары;
	- столбец date сохраняет момент создания нового отзыва;
	- сам отзыв (столбец review) ограничен определенным количеством символов. Для упрощения не вижу смысла добавлять возможность прикрепления картинки к отзыву.
8. Таблица orders:
	- хранит историю всех заказов.
9. Таблица orders_items:
	- хранит список купленных товаров с учетом id заказа;
	- дополнительно добавлены столбцы цены товара на момент покупки (price) и количество купленных товаров (quantity).
